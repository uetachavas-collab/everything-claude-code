# セキュリティガイドライン（日本チーム向け）

> [common/security.md](../common/security.md) の共通ルールを拡張した、日本の法規制対応を含む追加規約。

## 必須セキュリティチェック

コミット前に確認する。

- [ ] API キー・パスワード・トークンをコードにハードコードしていないか
- [ ] すべてのユーザー入力を検証しているか
- [ ] SQL インジェクション対策（プリペアードステートメント）ができているか
- [ ] XSS 対策（HTML エスケープ）ができているか
- [ ] CSRF 保護が有効か
- [ ] 認証・認可の確認ができているか
- [ ] 全エンドポイントにレート制限があるか
- [ ] エラーメッセージに機密情報が含まれていないか

## 個人情報保護法（PIPA）対応

日本の個人情報保護法に基づく対応が必要な箇所。

### 個人情報の定義

以下は個人情報として扱う：

- 氏名・住所・電話番号・メールアドレス
- マイナンバー（特定個人情報）
- 生体情報（顔画像・指紋等）
- 購買履歴・位置情報（特定の個人と紐付く場合）

### 実装上の注意点

```typescript
// NG: ログに個人情報を含める
logger.info(`ユーザー ${user.name} (${user.email}) がログイン`);

// OK: 識別子のみをログに記録する
logger.info(`ユーザーログイン`, { userId: user.id });

// NG: エラーレスポンスに個人情報を含める
return res.status(400).json({ error: `${user.email} は登録されていません` });

// OK: 汎用的なエラーメッセージを返す
return res.status(400).json({ error: 'メールアドレスまたはパスワードが正しくありません' });
```

### データ保持期間

個人情報を扱うデータには保持期間を設定する。

```typescript
const DATA_RETENTION = {
  USER_SESSION: 30 * 24 * 60 * 60, // 30日（秒）
  ACCESS_LOG: 365 * 24 * 60 * 60,  // 1年（法的要件に応じて）
  PAYMENT_RECORD: 7 * 365 * 24 * 60 * 60, // 7年（税法上の要件）
} as const;
```

## 機密情報の管理

- シークレット（API キー・パスワード）は環境変数または AWS Secrets Manager・GCP Secret Manager で管理する
- `.env` ファイルは `.gitignore` に含め、絶対にコミットしない
- 漏洩した可能性のあるシークレットは即座にローテーションする

## セキュリティインシデント対応

問題を発見したら：

1. **即座に作業を停止する**
2. セキュリティ担当者・チームリーダーに報告する
3. 影響範囲を調査する
4. 漏洩した可能性のある情報をローテーションする
5. 再発防止策をドキュメント化する

## 電子署名・タイムスタンプ

電子帳簿保存法に対応するシステムでは：

- タイムスタンプは信頼できる第三者機関（TSA）のものを使用する
- 電子署名は JIS X 5104 / JIS X 5092 に準拠した実装を検討する
