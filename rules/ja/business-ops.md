# 日本ビジネス業務ルール

日本のビジネス慣習・業務フローに関するガイドライン。

## 承認フロー（稟議）

システムに承認フローを実装する際は、日本の稟議文化を考慮する。

### 稟議の基本構造

```typescript
interface RingiRequest {
  id: string;
  title: string;           // 件名
  requester: string;       // 申請者
  requestDate: Date;       // 申請日
  amount?: number;         // 金額（費用稟議の場合）
  purpose: string;         // 目的・理由
  approvers: Approver[];   // 承認者リスト（順番に承認）
  status: RingiStatus;
}

type RingiStatus = 
  | 'draft'      // 起案中
  | 'pending'    // 承認待ち
  | 'approved'   // 承認済み
  | 'rejected'   // 否決
  | 'withdrawn'; // 取り下げ
```

### 承認の順序

- 承認は**順番**に行う（並行承認ではなく直列承認が基本）
- 上位の承認者が否決した場合、下位の承認は無効になる
- 差し戻しの場合は申請者に理由とともに返却する

チームコミュニケーションの基本。システムの通知設計に活かす。

| 種別 | タイミング | チャンネル |

## 会計・財務処理

### 消費税

```typescript
const TAX_RATES = {
  STANDARD: 0.10,    // 標準税率（10%）
  REDUCED: 0.08,     // 軽減税率（8%、食料品・新聞等）
} as const;

// 税込金額の計算
function calcTaxIncluded(price: number, rate = TAX_RATES.STANDARD): number {
  return Math.floor(price * (1 + rate));
}
```

### 請求書フォーマット（インボイス制度対応）

2023年10月から施行のインボイス制度（適格請求書等保存方式）に対応する。

必須記載項目：
- 適格請求書発行事業者の登録番号（T + 13桁）
- 課税資産の譲渡等を行った年月日
- 課税資産の譲渡等に係る資産・役務の内容
- 税率ごとの合計額および消費税額
- 書類の交付を受ける事業者の氏名・名称

## 個人情報の取り扱い

### 取得・利用

- 利用目的を明示してから個人情報を取得する
- 取得した目的以外での利用は禁止（法的例外あり）
- 個人情報の第三者提供は原則として本人同意が必要

### 安全管理措置

- 個人情報へのアクセスは**最小権限の原則**に従う
- アクセスログを記録・保持する（最低1年）
- 定期的にアクセス権限を棚卸しする

## 電子帳簿保存法（電帳法）対応

電子取引の記録を保存するシステムに必要な要件：

- 真実性の確保：タイムスタンプまたは訂正削除ができないシステムの利用
- 可視性の確保：検索機能（日付・金額・取引先で検索できること）
- 保存期間：法人は原則7年（欠損金が生じた年は10年）
