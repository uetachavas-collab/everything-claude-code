# 日本チーム向けルール

このディレクトリには、日本のビジネス慣習・法規制・開発文化に対応した追加ルールが含まれています。

## ファイル構成

| ファイル | 内容 |
|---|---|
| [coding-style.md](./coding-style.md) | コメント・命名規則・日付/数値フォーマット |
| [security.md](./security.md) | 個人情報保護法（PIPA）対応・セキュリティ実装 |
| [git-workflow.md](./git-workflow.md) | ブランチ命名・PR ルール・リリースフロー |
| [testing.md](./testing.md) | テスト記述・日本固有のテストケース（和暦・消費税等） |
| [business-ops.md](./business-ops.md) | 稟議フロー・報連相・インボイス制度・電帳法対応 |

## 使い方

これらのルールは [common/](../common/) の共通ルールを**拡張**するものです。共通ルールと矛盾する場合は、このディレクトリのルールを優先してください。

## 関連スキル

- `/japanese-business-ops` — 稟議・報連相ワークフロー
- `/jira` — Jira チケット操作
- `/backlog` — Backlog プロジェクト管理
