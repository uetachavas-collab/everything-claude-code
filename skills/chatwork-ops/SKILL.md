# Chatwork 操作スキル

Chatwork（日本発のビジネスチャットツール）との連携ワークフロー。

## 使いどき

- Chatwork でメッセージを送信・確認するとき
- グループルームにお知らせや報告を投稿するとき
- Chatwork タスクを追加・管理するとき
- 特定のルームのメッセージを検索するとき
- 日報・週報を定型フォーマットで送信するとき

## セットアップ

### MCP サーバーの設定

`~/.claude.json` の `mcpServers` に以下を追加する：

```json
"chatwork": {
  "command": "npx",
  "args": ["-y", "chatwork-mcp-server"],
  "env": {
    "CHATWORK_API_TOKEN": "あなたのAPIトークン"
  }
}
```

### API トークンの取得

1. Chatwork にログインする
2. 右上のアカウント名 → 「API トークン」をクリック
3. トークンを発行してコピーする

## 仕組み

### メッセージのフォーマット

Chatwork 独自のマークアップを使うと、メッセージが見やすくなる。

```
[info][title]件名[/title]
本文をここに書く
[/info]

[to:123456] 田中さん、確認をお願いします。

[qt][qtmeta aid=789 time=1234567890]
引用テキスト
[/qt]
```

### 日報・週報のテンプレート

```
[info][title]日報 2026年04月10日（木）[/title]
担当: 山田太郎

【本日の作業】
- PROJ-123 ログインバリデーション修正（完了）
- PROJ-124 検索APIの調査（進行中）

【明日の予定】
- PROJ-124 検索API実装
- 週次ミーティング参加

【共有事項】
なし
[/info]
```

### 障害報告のテンプレート

```
[info][title]【障害報告】本番環境でログインができない事象[/title]
発生時刻: 2026年04月10日 14:30
影響範囲: 全ユーザー
ステータス: 調査中

【状況】
本番サーバーのログインAPIが503を返している。

【対応中】
- サーバー担当: 田中がログ確認中
- 予想復旧時刻: 15:00

次回報告: 15分後
[/info]
[to:全員]
```

## 出力フォーマット

### ルーム一覧の出力例

```
## Chatwork ルーム一覧

| ルーム名 | 種別 | 未読数 | 最終メッセージ |
|---|---|---|---|
| 開発チーム | グループ | 3 | 山田: PROJ-123 修正しました |
| 経営陣 | グループ | 0 | - |
| 田中太郎 | DM | 1 | 明日の打ち合わせについて... |
```

### メッセージ送信の確認

```
以下の内容を「開発チーム」ルームに送信します：

[info][title]日報 2026年04月10日[/title]
...
[/info]

送信してよいですか？
```

## 使用例

```
Chatwork の開発チームルームに本日の日報を送ってください。
本日の作業: ログイン画面のバグ修正、APIテスト
明日の予定: パフォーマンス改善作業

Chatwork でルーム一覧を表示してください。

PROJ-123 の作業が完了したことを田中さんに Chatwork でお知らせしてください。
```
