# Backlog 連携スキル

Backlog（ヌーラボ社が提供する日本発のプロジェクト管理ツール）との連携ワークフロー。

## 使いどき

- Backlog の課題を検索・参照・作成・更新するとき
- バグ報告・機能要望を Backlog 課題として登録するとき
- マイルストーン・スプリントの進捗を確認するとき
- プロジェクトの Wiki を参照するとき
- 課題にコメントを追加するとき

## セットアップ

### MCP サーバーの設定

`~/.claude.json` の `mcpServers` に以下を追加する：

```json
"backlog": {
  "command": "npx",
  "args": ["-y", "backlog-mcp-server"],
  "env": {
    "BACKLOG_SPACE_ID": "あなたのスペースID",
    "BACKLOG_API_KEY": "あなたのAPIキー"
  }
}
```

### API キーの取得

1. Backlog にログインする
2. 個人設定 → API → API キーを発行する
3. キーをコピーして環境変数に設定する

## 仕組み

### 課題の検索

```
プロジェクトキー: PROJ
ステータス: 未対応、処理中、処理済み、完了
優先度: 高、中、低
担当者: ユーザー名
マイルストーン: マイルストーン名
```

### 課題の作成

Backlog 課題を作成するときに必要な情報：

| フィールド | 説明 | 必須 |
|---|---|---|
| 件名 | 課題のタイトル | ○ |
| 種別 | バグ・タスク・要望・その他 | ○ |
| 優先度 | 高・中・低 | ○ |
| 担当者 | 担当ユーザー | △ |
| マイルストーン | 対象マイルストーン | △ |
| カテゴリー | 分類タグ | △ |
| 期限日 | 完了予定日 | △ |
| 詳細 | 説明（Markdown 対応） | △ |

### 課題の命名規則

Backlog 課題は `プロジェクトキー-番号` で識別する（例: `PROJ-123`）。

コミットメッセージやブランチ名に課題番号を含める：

```bash
# ブランチ名
feature/PROJ-123-ユーザー認証機能

# コミットメッセージ
feat(auth): PROJ-123 ユーザー認証機能を実装
```

## 出力フォーマット

### 課題一覧の出力例

```
## Backlog 課題一覧（プロジェクト: PROJ）

### 処理中（3件）
- [PROJ-123] ログイン画面のバリデーションエラーを修正（担当: 山田）期限: 2026/04/15
- [PROJ-124] 商品検索 API のパフォーマンス改善（担当: 田中）期限: 2026/04/20
- [PROJ-125] 請求書 PDF 出力機能の追加（担当: 鈴木）期限: 2026/04/30

### 未対応（5件）
...
```

### 課題詳細の出力例

```
## PROJ-123: ログイン画面のバリデーションエラーを修正

**種別**: バグ
**優先度**: 高
**担当者**: 山田太郎
**マイルストーン**: v2.3.0
**期限**: 2026年04月15日
**ステータス**: 処理中

### 詳細

パスワードが8文字未満の場合にバリデーションエラーが表示されないバグが報告されています。

### 再現手順

1. ログイン画面を開く
2. パスワードフィールドに5文字を入力
3. 送信ボタンをクリック
4. エラーメッセージが表示されない
```
