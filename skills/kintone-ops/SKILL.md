# kintone 操作スキル

kintone（サイボウズ社が提供する日本発のノーコード業務アプリプラットフォーム）との連携ワークフロー。

## 使いどき

- kintone の顧客管理・案件管理アプリのレコードを参照・更新するとき
- 業務データを kintone から取得してレポートを作成するとき
- kintone アプリに新規レコードを追加するとき
- 複数のアプリをまたいでデータを集計するとき
- kintone のフォームデータを分析するとき

## セットアップ

### MCP サーバーの設定

`~/.claude.json` の `mcpServers` に以下を追加する：

```json
"kintone": {
  "command": "npx",
  "args": ["-y", "@kintone/mcp-server"],
  "env": {
    "KINTONE_BASE_URL": "https://あなたのサブドメイン.cybozu.com",
    "KINTONE_USERNAME": "あなたのユーザー名",
    "KINTONE_PASSWORD": "あなたのパスワード"
  }
}
```

または API トークン認証を使う場合：

```json
"kintone": {
  "command": "npx",
  "args": ["-y", "@kintone/mcp-server"],
  "env": {
    "KINTONE_BASE_URL": "https://あなたのサブドメイン.cybozu.com",
    "KINTONE_API_TOKEN": "あなたのAPIトークン"
  }
}
```

### API トークンの取得

1. kintone にログインする
2. 対象アプリを開く
3. アプリ設定 → API トークン → 追加
4. 必要な権限（閲覧・追加・編集）を設定してトークンを生成する

## 仕組み

### よく使うアプリの種別

| アプリ種別 | 用途 |
|---|---|
| 顧客管理（CRM） | 取引先・担当者の情報管理 |
| 案件管理 | 営業案件の進捗管理 |
| 勤怠管理 | 出退勤・有給・残業の管理 |
| 経費精算 | 経費申請・承認フロー |
| プロジェクト管理 | タスク・マイルストーン管理 |
| 問い合わせ管理 | カスタマーサポートのチケット |

### クエリの書き方

kintone ではレコードの絞り込みに独自のクエリ構文を使う。

```
# 担当者が「山田」で、ステータスが「対応中」のレコードを取得
担当者 = "山田" and ステータス in ("対応中")

# 今月中に期限があるレコードを取得
期限日 >= LAST_MONTH() and 期限日 <= THIS_MONTH()

# 金額が100万円以上の案件を取得
金額 >= 1000000
```

## 使用例

```
kintone の案件管理アプリで、今月期限の案件を一覧表示してください。

kintone の顧客管理アプリで「株式会社A」のレコードを検索してください。

kintone の問い合わせ管理アプリに新規レコードを追加してください。
内容: Webサイトのフォームから問い合わせがあった件
担当: 田中花子
ステータス: 未対応
```
