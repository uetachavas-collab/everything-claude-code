# ドキュメント参照（Context7）

ライブラリ・フレームワーク・API について質問された場合、トレーニングデータに頼るのではなく Context7 MCP ツール（`resolve-library-id` と `query-docs`）経由で最新ドキュメントを取得して回答する。

## 核心概念

- **Context7**: ライブドキュメントを公開する MCP サーバー。ライブラリと API のトレーニングデータの代わりに使用する。
- **resolve-library-id**: ライブラリ名とクエリから Context7 互換のライブラリ ID（例: `/vercel/next.js`）を返す。
- **query-docs**: 指定されたライブラリ ID と質問に対してドキュメントとコードスニペットを取得する。必ず先に resolve-library-id を呼んで有効なライブラリ ID を取得すること。

## 使いどき

以下の場合に有効化する：

- セットアップや設定に関する質問（例: 「Next.js のミドルウェアはどう設定する?」）
- ライブラリに依存するコードのリクエスト（「Prisma で〜のクエリを書いて」）
- API やリファレンス情報が必要（「Supabase の認証メソッドは?」）
- 特定のフレームワークやライブラリが言及された（React・Vue・Svelte・Express・Tailwind・Prisma・Supabase など）

ライブラリ・フレームワーク・API の正確で最新の動作に依存するリクエストに対して使用する。Context7 MCP が設定されたハーネス（Claude Code・Cursor・Codex など）に適用される。

## 仕組み

### ステップ 1: ライブラリ ID を解決する

**resolve-library-id** MCP ツールを以下で呼び出す：

- **libraryName**: ユーザーの質問からのライブラリまたは製品名（例: `Next.js`・`Prisma`・`Supabase`）。
- **query**: ユーザーの質問全文。関連性ランキングを向上させる。

このステップから Context7 互換のライブラリ ID（形式: `/org/project` または `/org/project/version`）を取得してから、query-docs を呼ぶこと。

### ステップ 2: 最適なマッチを選択する

解決結果から以下を使って1つ選択する：

- **名前の一致**: ユーザーが求めたものに完全または最も近いマッチを優先。
- **ベンチマークスコア**: 高いスコアはドキュメント品質が良いことを示す（100が最高）。
- **バージョン**: ユーザーがバージョン（例: 「React 19」「Next.js 15」）を指定した場合、バージョン固有のライブラリ ID を優先。

### ステップ 3: ドキュメントを取得する

**query-docs** MCP ツールを以下で呼び出す：

- **libraryId**: ステップ 2 で選択した Context7 ライブラリ ID（例: `/vercel/next.js`）。
- **query**: ユーザーの具体的な質問またはタスク。関連するスニペットを得るため具体的に。

制限: 1質問あたり query-docs（と resolve-library-id）の合計呼び出しを3回以内にする。

### ステップ 4: ドキュメントを使用する

- 取得した最新情報を使ってユーザーの質問に回答する。
- 役立つ場合はドキュメントからの関連コード例を含める。
- 重要な場合はライブラリまたはバージョンを引用する（例: 「Next.js 15 では...」）。

## ベストプラクティス

- **具体的に**: 可能な限りユーザーの質問全文をクエリとして使用する。
- **バージョン認識**: ユーザーがバージョンを指定した場合、resolve ステップからのバージョン固有のライブラリ ID を使用する。
- **公式ソースを優先**: 複数のマッチが存在する場合、コミュニティフォークより公式または主要パッケージを優先する。
- **機密データを含めない**: Context7 に送信するクエリから API キー・パスワード・トークンなどの秘密情報を削除する。
